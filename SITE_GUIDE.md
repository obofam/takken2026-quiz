## デプロイ方法（Vercel CLI）

**前提：** Node.js が入っていること。初回のみ `npm install -g vercel` でCLIを入れる。

### 毎回の更新手順

1. ターミナルで `7_クイズと進捗トラッカー` フォルダに入る（⚠旧 `有料PDF\quiz_site` は2026-06-08に廃止・アーカイブ済み）
   ```
   cd C:\Claude\宅建\7_クイズと進捗トラッカー
   ```

2. （初回or未ログイン時のみ）Vercelにログイン
   ```
   vercel login
   ```
   → メールアドレス入力 → 届いたメールの「Verify」クリック

3. （初回のみ）既存プロジェクトに紐付け
   ```
   vercel link
   ```
   - `Set up "~/quiz_site"?` → **Y**
   - `Which scope?` → **obocks-projects** を選ぶ
   - `Link to existing project?` → **Y**
   - `What's the name of your existing project?` → **takken2026-quiz**
   - `? Would you like to pull environment variables now?` → **N**（静的HTMLなので不要）

   → `.vercel` フォルダができて紐付け完了。これ以降は毎回不要。

4. **本番デプロイ**
   ```
   vercel --prod
   ```
   数十秒でビルド→ https://takken2026-quiz.vercel.app/ に反映。

### トラブル時

- `vercel link` をやり直したい → `7_クイズと進捗トラッカー` 内の `.vercel` フォルダを削除してから再実行
- ログインし直し → `vercel logout` → `vercel login`
- 何がデプロイされてるか確認 → `vercel ls`

### 更新チェックリスト

- [ ] `index.html` を編集したあと、ブラウザでローカル確認（ダブルクリックで開く）

## 設計思想

### コアコンセプト
「PDFは基礎インプット＋初回の力試し、Webは反復学習の場」という役割分担。

### サイト構造（3階層）

```
[TOP]
  ├─ 科目カード一覧（ロック中/解除済み/近日公開）
  └─ 解除済み科目には 挑戦数/正答率/ベスト を表示
        ↓ タップ
[科目詳細]
  ├─ 大きめの数字表示（挑戦数/正答率/ベスト）
  └─ 4つのモード選択
        ↓ タップ
[クイズ]
  ├─ 進捗バー＋カウンター
  ├─ ○×ボタン（老眼対応で2.4rem大）
  └─ 即時フィードバック＋解説
```

### 4つの学習モード

| モード | 目的 | 採点 |
|---|---|---|
| 全問チャレンジ | 初見・実力測定 | ◎（統計に反映） |
| ランダム10問 | スキマ時間の小テスト | ◎ |
| 間違えた問題だけ | ピンポイント復習 | ◎ |
| 全問レビュー | 解説通読（インプット） | × |

### 「2回連続正解で卒業」システム

#### 背景
「1回正解で間違い履歴から消す」方式は、まぐれ正解や直前記憶を実力と誤認するリスクがあり、特に数字・要件を正確に覚える宅建には不向き。

#### ルール
- 間違えると `missCount++`、`consecCorrect=0`、履歴入り
- 正解1回目：「あと1回で卒業」表示、履歴に残る
- 正解2回目（連続）：履歴から削除「◎卒業！」
- 間違えるとカウントリセット

#### UI表示
各問題カードの上部に状態タグを表示：
- 赤系「ミス3回」
- 緑系「あと1回で卒業」

### データ永続化（localStorage）

キー：`takken_quiz_v1`

```json
{
  "unlocked": { "gyouhou": true },
  "stats": {
    "gyouhou": { "attempts": 5, "bestPct": 85, "totalCorrect": 120, "totalAnswered": 150 }
  },
  "mistakes": {
    "gyouhou": { "7": { "missCount": 2, "consecCorrect": 1 } }
  }
}
```

統計カウント対象：
- 全問チャレンジ / ランダム10問 / 間違えた問題だけ → 挑戦数に加算
- 全問レビュー → 記録なし
